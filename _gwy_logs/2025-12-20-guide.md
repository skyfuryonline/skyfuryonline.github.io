### 欢迎开启 2026 备考之旅！

这是您的备考记录系统 **2.0 版本** 的完整使用说明书。本系统旨在通过**精细化的数据统计**和**可视化的反馈**，辅助您科学备考。

---

#### 1. 如何记录？(核心)

您只需在 `_gwy_logs` 目录下创建一个新的 `.md` 文件（建议用日期命名），并填入以下内容。

**推荐模板 (直接复制)**:

```yaml
---
date: 2026-03-15 20:00:00
title: "今天攻克了图形推理"
subjects:
  - name: 资料分析
    hours: 2
  - name: 判断推理
    hours: 3.5
  - name: 言语理解
    hours: 0
  - name: 数量关系
    hours: 0
  - name: 申论
    hours: 1.5
  - name: 其他
    hours: 0
tags: [刷题, 复盘]
---

### 📝 今日复盘

今天重点复习了图形推理，发现自己对“一笔画”问题还是不够敏感...
```

**字段说明**:
*   **subjects**: 這是核心统计源。请在对应的科目下填写小时数。
*   **tags**: 仅作为标记（如“模考”、“听课”），**不**参与时长统计。

---

#### 2. 智能归类与映射系统

为了方便记录，系统内置了**关键词映射**功能。您在 `name` 字段中可以使用简称，系统会自动归类到标准大类。

| 您填写的 (简称) | 系统归类为 (标准大类) | 饼图颜色 |
| :--- | :--- | :--- |
| **资料**、速算 | **资料分析** | 🔵 蓝色 |
| **图推**、逻辑、类比 | **判断推理** | 🟣 紫色 |
| **言语**、片段阅读 | **言语理解** | 🟢 绿色 |
| **数学**、数量 | **数量关系** | 🟠 橙色 |
| **常识** | **常识判断** | 🟢 青色 |
| **大作文**、公文 | **申论** | 🔴 红色 |
| 面试、模拟 | 面试/模考 | 🟡 黄色/深蓝 |

*建议：尽量使用标准大类名称，或者固定的简称，以保持习惯一致。*

---

#### 3. 可视化图表解读

*   **月历卡片 (中间)**: 
    *   显示当月的打卡情况。
    *   **颜色深浅**代表当天的总学习时长 (0h 灰色 -> 6h+ 深绿)。
    *   点击月份两侧的箭头 `< >` 可以切换月份。
*   **最近7天趋势 (左侧上)**: 
    *   显示过去7天的总学习时长柱状图。
*   **科目分布 (左侧下)**: 
    *   显示本周（或根据日志累计）的各科目投入比例。
    *   颜色与上面的表格对应。

### 如何配置 AI 周报？

本系统支持每周自动调用大语言模型（LLM），分析您一周的学习记录和日记，生成一份专业的周报。要启用此功能，您需要完成以下配置：

#### 1. 在 GitHub 设置 Secrets

为了安全地存储您的 API 密钥，请在您的 GitHub 仓库中进行如下设置：

1.  进入仓库主页，点击 **Settings** -> **Secrets and variables** -> **Actions**。
2.  点击 **New repository secret** 按钮。
3.  **Name**: `LLM_API_KEY`
4.  **Secret**: 粘贴您的 API 密钥（例如，OpenAI 的 `sk-...`）。
5.  **(可选)** 如果您使用代理服务，请再添加一个 Secret：
    *   **Name**: `LLM_API_BASE_URL`
    *   **Secret**: 您的代理 URL。

#### 2. 修改 `weekly_report.yml` 工作流

打开 `.github/workflows/weekly_report.yml` 文件，找到 `Generate Weekly Report` 这个步骤，并修改 `env` 部分，使其能读取您设置的 Secret。

**修改前**:
```yaml
env:
  # 您需要在这里设置您的 LLM API 密钥
  # OPENAI_API_KEY: ${{ secrets.YOUR_LLM_API_KEY_SECRET }} 
  # ...
```

**修改后 (以 OpenAI 为例)**:
```yaml
env:
  OPENAI_API_KEY: ${{ secrets.LLM_API_KEY }}
  # 如果您用了代理，也取消这一行的注释
  # OPENAI_API_BASE: ${{ secrets.LLM_API_BASE_URL }}
```

#### 3. 修改 `weekly_summary_generator.py` (激活真实调用)

打开 `crawlers/weekly_summary_generator.py`，找到 `get_llm_summary` 函数。

1.  **取消真实调用的注释**: 找到 `response = client.chat.completions.create(...)` 和 `summary = response.choices[0].message.content` 这几行，并取消它们的注释。
2.  **删除或注释掉示例周报**: 找到 `summary = """..."""` 这个大的文本块，并将其删除或用 `"""` 注释掉。
3.  **(可选)** 您也可以在这里修改想用的模型，例如 `model="gpt-4o"`。

完成以上步骤后，每周日的自动周报功能就会正式启用。
